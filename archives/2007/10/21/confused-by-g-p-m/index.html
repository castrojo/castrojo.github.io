    Confused by G-p-m

    I bought a new laptop, and I wanted to set the brightness settings between AC and battery power.

    I was seriously confused, because my display kept dimming when I didn’t think it should. As it ends out, when on AC, gnome-power-manager has a slider for the display, which reads “Set Display Brightness to:” and then a slider for 0-100%

    [](http://stompbox.typepad.com/photos/uncategorized/2007/10/21/ac.png)[

    Unfortunately, when on battery, the same setting in the battery tab for display brightness reads as “Dim display brightness by:” then a slider for 0-100%

    [](http://stompbox.typepad.com/photos/uncategorized/2007/10/21/battery.png)[

    The sliders aren’t consistent with each other. (I am referring to the last slider in each pic)

    One measures brightness, the other measures “dimness”. I want to set one for AC, and one for battery. I shouldn’t have to measure brightness for one, and “dimness” for the other, they should use the same “scale”. I don’t know if brightness or “dimness” is the right decision, I’m just saying that they should be consistent with each other within the preferences dialog.

    Unfortunately there are issues with GNOME’s bugzilla at the moment, so I can’t really file a bug; I will do so first thing in the morning on Monday.

    Anyone else get confused by these sliders or report this bug already? I was convinced that this was a problem with my laptop until my friend Rick Harding pointed it out to me that the sliders measured totally different things. I hadn’t even noticed that they were different, as it ends up, they were doing exactly what I had set them to do.
