<div class="entry-content" itemprop="articleBody">

I was at lunch the other day and [Ken Simon](http://ninkendo.org), a
sysadmin at [Oakland University](http://www.oakland.edu). He was
mentioning how the students were on spring break and he needed a quick
and dirty project to work on.

So I told him that he could probably implement the
[blueprint](https://wiki.ubuntu.com/GnomeDo/Specs/Launchpad) I had
written a month or so ago for a Launchpad plugin for GNOME Do. I wrote
the spec because as I use Do more and more I tend to want everything I
work in to just be in Do because it’s a fast way to work.

As Ken [explains](http://ninkendo.org/blog/byentryid/47), a few hours
later he had something working. Behold:

[![](http://lh6.google.com/jorge.castro/R8iA_9wSr9I/AAAAAAAAEog/0DmgV-YUsKg/s400/do-lp-plugin.png)](http://ethos.battleaxe.net/~jorge/launchpad-do.ogg)

Do is one of those applications that you need to see to really “get it”.
So I’ve put together a screencast of Ken’s [Launchpad Plugin for GNOME
Do](http://ethos.battleaxe.net/~jorge/launchpad-do.ogg) (Ogg Video) so
you can check out how it works.

We will be merging this plugin into the official plugins branch, but in
the meantime you can grab the code
[here](https://code.edge.launchpad.net/~ninkendo/do/do-plugins-launchpad).
If you don’t want to build from source, snag the .dll from
[here](http://ethos.battleaxe.net/~jorge/Launchpad.dll) and copy it into
\~/.local/share/gnome-do/plugins then restart GNOME Do. You need the
version of GNOME Do from the official [Personal Package
Archive](https://edge.launchpad.net/~do-core/+archive).

Also, GNOME Do now has *learning* enabled in trunk. This means that as
you use Do, over time it learns what things you tend to use the most. So
if you keep using “Launchpad -&gt; 123456 -&gt; Bug Number” then the Bug
Number item will start to drift to the top of the list. If your friend
is a translator and keeps doing “Launchpad -&gt; Russian -&gt; Release
Translations” then the translation item will start to move to the top of
his list. As you can imagine, after Do learns the way you work, you
become real fast at getting to where you want to be.

Thanks to Ken Simon for implementing this, and David Siegel for
answering his questions on the list. I am finding myself more efficient
with this plugin and the learning feature. Enjoy!
