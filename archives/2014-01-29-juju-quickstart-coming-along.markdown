<div class="entry-content" itemprop="articleBody">

One of the more annoying things about setting up cloud tools is copying
and pasting your auth keys and strings into config files. Some people
set variables, other have config files.

One thing we wanted to do in Juju is to make sure that people can
generate the proper configs for their clouds, and that can be
problematic with people copying and pasting snippets around, so behold,
`juju quickstart`.

When invoked with `juju quickstart -i` we offer you fields to fill in
for whatever you are deploying on:

![](//ubuntu-discourse.s3.amazonaws.com/752c64f3a54c149c55c914d6c14ed7348cbd6727b94.png){width="690"
height="441"}

And here’s the fields for a provider:

![](//ubuntu-discourse.s3.amazonaws.com/7538343db659b8d7d3c12c7ab0a4bf1589e6ae7a8a0.png){width="690"
height="441"}

And here’s my favorite part, when a field is wrong or missing, it tells
you!

![](//ubuntu-discourse.s3.amazonaws.com/75450afe5f9e0a38cc3b6156f48e6cea51edef58ecb.png){width="690"
height="475"}

This should help making it easier to get up and running.
