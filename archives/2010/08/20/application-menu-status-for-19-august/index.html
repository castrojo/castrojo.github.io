    Application Menu Status for 19 August

    Hear ye, hear ye,

    It’s been a while since the last application menu status. Since the last one the app menu developers have had a sprint, a bunch of other work in Unity, including some holidays as well as a new child process in Ted’s case, so now things can settle and be back to “normal”:

    Here are the changelogs:
    [https://code.launchpad.net/~indicator-applet-developers/indicator-appmenu/trunk](https://code.launchpad.net/~indicator-applet-developers/indicator-appmenu/trunk)
    [https://code.launchpad.net/~canonical-dx-team/appmenu-gtk/trunk](https://code.launchpad.net/~canonical-dx-team/appmenu-gtk/trunk)

    Most of the work is now feature complete, here are the bugs which are currently “hot” on the radar for the team which have the most priority:

    • [https://bugs.launchpad.net/appmenu-gtk/+bug/604668](https://bugs.launchpad.net/appmenu-gtk/+bug/604668) - extra registrations create applications slowness

    • [https://bugs.launchpad.net/appmenu-gtk/+bug/611636](https://bugs.launchpad.net/appmenu-gtk/+bug/611636) - Alt-shortcuts display submenus of the local menu bar (this is the famous “shows in the app instead of the top bar” bug)

    • [https://bugs.launchpad.net/indicator-appmenu/+bug/607114](https://bugs.launchpad.net/indicator-appmenu/+bug/607114) - Supportmenu activation via the keyboard

    • [https://bugs.launchpad.net/indicator-appmenu/+bug/591186](https://bugs.launchpad.net/indicator-appmenu/+bug/591186) - Present aspecial menu for the desktop

    • [https://bugs.launchpad.net/indicator-appmenu/+bug/591188](https://bugs.launchpad.net/indicator-appmenu/+bug/591188) - Manageunresponsive applications

    As always, if you see an issue that you feel needs extra love please
    feel free to ping myself, dbarth, or seb128.
